# Система бронювання квитків
## Опис предметної області
Cистема бронювання квитків дозволяє користувачам шукати доступні рейси, обирати потрібний варіант та створювати бронювання. Адміністратор може керувати рейсами та розкладом. Після підтвердження та оплати бронювання користувач отримує електронний квиток. Система зберігає інформацію про рейси, користувачів, бронювання та квитки.

## Шарова архітектура 

Presentation Layer (UI)
Відповідає за взаємодію з користувачем: пошук рейсів, вибір місця, перегляд квитків.

Business Logic Layer (Service)
Містить логіку бронювання, перевірку доступності місць, генерацію квитків та обробку платежів.

Data Access Layer (Repository)
Забезпечує доступ до бази даних, роботу з таблицями користувачів, рейсів, бронювань і квитків.

## Клієнт–серверна архітектура

Клієнт надсилає запити (наприклад, пошук рейсів або створення бронювання), а сервер обробляє ці запити та повертає результати.
